# GSD Research Summary

## Stack Recommendations
- **Framework:** React Native / Expo (Best for fast iterative mobile UI for iOS + Android).
- **Animations:** React Native Reanimated (For drawing pattern lines and pulsing dots).
- **Haptics:** expo-haptics (Crucial for physical feedback during pattern drawing).
- **Storage:** @react-native-async-storage/async-storage (For cycle tracking offline).
- **Notifications:** expo-notifications (Local scheduling based on female cycle timing).

## Core Features (Table Stakes)
- **Pattern Lock Interface:** Simulated 3x3 dot lockscreen canvas.
- **Educational Patterns:** Predefined shapes simulating the 3 auto-exam touch modes (Radial, Spiral, Vertical).
- **Feedback Loop:** Success/Failure glow & haptic vibration upon completing a drawn pattern.
- **Reminder Engine:** Local scheduled alarms prompting usage 1x a month.

## Architecture
- **UI Canvas Layer:** Renders the 9 points, tracks panning gestures using PanGestureHandler, draws SVG lines/animated views.
- **Rules Engine:** Validates if drawn pattern matches the target educational motion.
- **State/Cycle Manager:** Keeps the monthly timestamp and schedules OS-level notifications.

## Key Pitfalls (Critical)
1. **OS Modification Ban:** Reemplazar el Lock Screen nativo es imposible. Solución: Usar notificaciones ricas tipo "Alarma mensual" o Vault.
2. **Hitbox Precisión:** Los puntos deben tener buen "hit slop" para evitar fricción en el aprendizaje.
3. **Privacidad:** Los datos del ciclo (si se piden) deben ser estrictamente locales.
